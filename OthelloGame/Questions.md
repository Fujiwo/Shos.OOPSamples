# オセロゲーム - 問題

## 問題1: オセロゲームのクラス設計

プレイヤー対CPUで遊べるオセロゲーム（リバーシ）を作成します。以下の要件を満たすクラス設計を考えてください。

### ゲームのルール

1. 8×8のボード上で、黒と白の石を使って対戦します
2. 相手の石を挟むと、挟まれた石は自分の色に変わります
3. 石を置ける場所がない場合はパスとなります
4. 両者とも石を置けなくなった時点でゲーム終了、石の数が多い方が勝ちです

### 要件

1. **ボード (Board)**: 8×8のマス目を管理します
2. **石 (Disc/Piece)**: 黒または白の状態を持ちます
3. **プレイヤー (Player)**: 人間プレイヤーとCPUプレイヤーがいます
4. **ゲーム管理 (Game)**: ゲーム全体の進行を制御します
5. **表示/入力 (View)**: ボードの表示やユーザー入力を処理します

### 課題

1. 上記の要件を満たすクラス図を作成してください
2. クラス間の関係（継承、コンポジション、集約など）を明確にしてください
3. 各クラスの主要なプロパティとメソッドを定義してください

### ヒント

- 人間プレイヤーとCPUプレイヤーの共通部分は何でしょうか？
- ボードと石の関係はどのように表現できるでしょうか？
- ゲームのルール判定（合法手、石の反転）はどのクラスが担当すべきでしょうか？
- Model（ゲームロジック）とView（表示・入力）を分離するにはどうすればよいでしょうか？

## 問題2: プログラムの実装

上記で設計したクラス図を基に、以下の機能を持つプログラムを実装してください。

### 実装する機能

1. ゲームの初期配置（中央に黒白2個ずつ）
2. プレイヤーとCPUの交互の手番
3. 合法手の判定と表示
4. 石を置いたときの反転処理
5. ゲーム終了判定と勝敗の表示
6. CPUの思考（簡易的なアルゴリズムでOK）

### 実装言語

- Python
- HTML/CSS/JavaScript

### 注意点

- コードはできるだけシンプルに保ってください
- オブジェクト指向の原則に従ってください
- 各クラスの責務を明確にしてください
- Model（ゲームロジック）とView（表示・入力）を分離してください
